Architecture Feedback

There are a few things I would refine.

1. Add a search service abstraction

Right now search is defined as:

Postgres FTS or Meilisearch

But search will become critical.

Recommendation:

Create a SearchProvider interface.

Example:

SearchProvider

Implementations:

Postgres FTS (MVP)
Meilisearch (V1)
OpenSearch / Elasticsearch (V2)

Search engines eventually require separate scaling.

2. Preview pipeline architecture

The preview pipeline is well designed but needs one additional layer.

Right now it converts documents → PDF.

But you will also need:

thumbnail generation
page extraction
watermark overlay
text extraction

I would structure the pipeline as:

Upload
→ Scan
→ Convert
→ Extract text
→ Generate thumbnails
→ Index

Each stage should be a job.

3. Add a dedicated permission engine

Permissions are currently scattered across:

roles
groups
ACL
links

You should centralize evaluation into one module:

PermissionEngine

Example function:

canUserAccessDocument(user, document, action)

Actions:

view
download
print
share
comment

This prevents permission logic duplication.

4. Multi-tenant architecture decision

The document currently implies single organization per deployment.

But eventually users will want:

multi-tenant SaaS

So decide early if your database model supports:

organization_id
room_id

Every entity should include:

organization_id

Even if self-hosted installs only have one.
